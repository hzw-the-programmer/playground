这个宏用了哪些申明宏的技术

---

在分析 `futures-util` 内部的 `delegate_all!` 宏时，它所使用的技术全部是**声明宏（`macro_rules!`）**领域的经典技巧。这个宏之所以强大，是因为它充分利用了声明宏的元编程能力来解析一个结构体定义，并自动生成大量实现代码。

具体来说，它涉及到以下核心技术：

# 1. 深度模式匹配（Pattern Matching）

声明宏可以匹配复杂的语法树。`delegate_all!` 必须能够解析结构体的完整定义，包括：

- 属性（如 `#[pin_project::pin_project]`）

- 可见性（`pub`）

- 泛型参数（包含生命周期、类型参数、约束）

- 结构体字段（包括字段属性、可见性、名称、类型）

它会使用类似这样的捕获模式：

```rust
(
    $(#[$meta:meta])*
    $vis:vis struct $name:ident $( < $($gen:ident $(: $bound:path)?),* $(,)? > )?
    $(where $($where_clause:tt)*)? 
    {
        $(
            $(#[$field_meta:meta])*
            $field_vis:vis $field:ident : $field_ty:ty
        ),* $(,)?
    }
)
```

这需要用到 `:meta`、`:vis`、`:ident`、`:ty` 等多种片段分类符，以及重复模式 `$(...)*` 来处理列表。

# 2. 重复模式与展开（`$()*` 与 `$()+`）

这是声明宏最基础也最关键的特性。在宏内部：

- 捕获字段列表时，使用 `$( $field:ident : $ty:ty ),*` 来匹配零个或多个逗号分隔的字段。

- 在展开代码时，使用 `$( 生成的代码 )*` 来为每个字段生成实现。

例如，为每个合适的字段调用原子宏：

```rust
$($crate::delegate_stream!($field);)*
```

这一行会为每个字段生成一个 `delegate_stream!` 调用，是代码批量生成的核心。

# 3. 宏调用链（Macro Chaining）

`delegate_all!` 本身并不直接生成最终的 impl 代码，而是将工作委托给一组更专用的内部原子宏。它是“总调度器”：

- 解析结构体，提取出字段信息。

- 然后为每个字段生成对 `$crate::delegate_stream!($field)`、`$crate::delegate_sink!($field, $item)` 等的调用。

- 这些原子宏再各自负责生成一个特定 `trait` 的实现。

这体现了声明宏的组合性——复杂的宏可以通过调用更简单的宏来构建，避免单个宏过于庞大。

# 4. $crate 路径前缀

所有对内部原子宏的调用都使用 `$crate::delegate_stream!` 这样的路径，而不是直接写 `delegate_stream!`。这保证当 `futures-util` 被依赖时，宏展开的代码能正确引用到自身 crate 内部的宏，避免命名冲突。

# 5. 利用 tt（Token Tree）参数传递

虽然 `:$ty` 可以捕获类型，但有时为了保留完整的语法信息（例如 `impl Stream` 这种非标准路径），宏会使用 `:tt` 来捕获任意 token 树并透传给下层宏。原子宏可能接收到一个 `tt` 参数（比如字段名或类型），再在内部进行二次解析。

# 6. 条件行为模拟

纯声明宏不能运行时检查 trait 是否实现，也不能在编译时做复杂的布尔判断。但 `delegate_all!` 通过“信任约束”的方式实现了条件生成：

- 原子宏生成的 `impl` 代码会明确写出 trait 约束，例如 `impl<St: Stream> Stream for MyStream<St>`。

- 如果内部字段不满足约束，编译器在后续使用时会报错。

- 因此，宏本身不需要“判断”；它只是无条件地生成代码，把类型检查留给编译器。

# 7. 利用属性传递

在解析结构体定义时，宏会原样保留属性（如 `#[pin_project::pin_project]` 和字段上的 `#[pin]`），并传递给 `pin_project` 这类过程宏协同工作。声明宏负责结构体的骨架生成，而过程宏负责 `Pin` 投影等高级转化。

---

# 💎 总结

`delegate_all!` 本质是一个精巧的声明宏调度器，它用到的关键技术包括：

- **深层模式匹配**解析 Rust 结构体语法

- **重复操作符** `$()*` / `$()+` 批量生成代码

- **宏调用链**将任务分包给原子宏

- `$crate` **前缀**保证路径正确

- `tt` **透传**保持语法灵活性

- **依靠编译器约束**实现“条件生成”的假象

这些技术组合在一起，实现了用极短的声明宏消除大型样板代码的目标。
