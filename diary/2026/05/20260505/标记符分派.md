@trait叫什么技术

---

你提供的这个宏片段，展示的是声明宏中一种经典的**标记符分派技术**（或叫 **@ 标记模式 / 内部规则模式**）。

它的核心思路是：把一个宏作为内部实现入口，用特殊的标识符（比如 `@trait`）作为第一token，来区分不同的“内部规则分支”，从而模拟出某种程度的多态或内部调用。

---

# 1. 具体形式分析

假设外层宏叫 `impl_future!`，定义大致如下：

```rust
macro_rules! impl_future {
    // 对外公开的入口，最终会转调内部标记规则
    ($name:ident < $($arg:ident),* >, $t:ty $(, $($bound:tt)*)?) => {
        impl_future!(@trait Future $name < $($arg),* > ($t) $(where $($bound)*)*);
    };

    // 内部规则：以 @trait 开头
    (@trait Future $name:ident < $($arg:ident),* > ($t:ty) $(where $($bound:tt)*)*) => {
        impl<$($arg),*> futures_core::future::Future for $name<$($arg),*>
        where $t: futures_core::future::Future $(, $($bound)*)* {
            type Output = <$t as futures_core::future::Future>::Output;
            delegate_future!(inner);
        }
    };
}
```

使用时，开发者只需要调用 `impl_future!(MyType<A, B>, InnerType, A: Debug, B: Send)`，宏会自动转发到 `@trait` 内部规则进行真正的代码生成。

---

# 2. 这个技术的核心命名

- **标记符分派 (Tag Dispatching)** — 利用方言化的开头 token (`@trait`) 充当“标签”来区分不同内部逻辑。

- **@ 标记模式** — 因习惯以 @ 开头而得名，在 Rust 声明宏设计中非常流行（`futures`、`tokio`、`pin_project` 中大量使用）。

- **内部规则 (Internal Rules)** — 表示这些分支不是给宏的使用者调用的，而是宏内部用来转发和递归的“私有分支”。

# 3. 为什么需要使用 @trait 这种技术？

直接写一套模式匹配可能会变得非常复杂，尤其是当宏需要支持多种 trait 时（同时实现 `Future`、`Stream`、`Sink` 等）。引入 `@trait` 类似“函数名”，可以把大宏拆分成多个内部专用分支，各自负责一种 trait 的实现，而外层入口只需要分发到合适的 `@` 标签即可。

这样做的好处：

- **可读性强**：清楚表示这是一条内部实现路径。

- **避免匹配冲突**：因为 `@trait` 是凭空创造的 token，不太可能与用户提供的 Rust 语法冲突。

- **便于扩展**：后续要加 `Stream` 实现，只需新增 `(@trait Stream ...) => { ... }` 内部规则，入口分支分发一下即可。

---

# 4. 类似的名称归纳

称呼	侧重点
标记符分派	强调用特殊标记来调度不同的宏分支
@ 标记模式	聚焦 `@` 前缀的约定
内部宏/内部规则	强调该分支仅用于内部实现，不对外
辅助宏辅助模式	描述它是主逻辑的辅助设施

在 Rust 官方参考和社区讨论中，最常见和准确的叫法是 **“内部规则”**（internal rules），而所用的 `@trait` 则称为 **“内部标记”**（internal marker）。
