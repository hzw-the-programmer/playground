```rust
async {
    let mut reader = Cursor::new([1, 2, 3, 4]);
    let mut output = [0u8; 4];
    reader.read_exact(&mut output).await?;
}
```

output 这个局部变量不直接在传统的调用栈上，它会被编译器提升到异步块生成的匿名 Future 状态中。至于这个 Future 状态最终在内存的哪里（栈还是堆），取决于你如何使用这个异步块。

# 🧩 异步块编译为状态机

```rust
async {
    let mut reader = Cursor::new([1, 2, 3, 4]);
    let mut output = [0u8; 4];
    reader.read_exact(&mut output).await?;
}
```

这段代码会被 Rust 编译器转换为一个状态机枚举，它的大致结构如下（示意，非真实代码）：

```rust
enum MyAsyncBlock {
    Start,
    AwaitingRead {
        reader: Cursor<[u8; 4]>,
        output: [u8; 4],
        // ... 存储 read_exact 返回的 Future 等
    },
    Done,
}
```

可以看到，output 和 reader 一起，成了枚举变体的字段，而不是在栈上独立存在的局部变量。所有跨 .await 点的局部变量都会被这样存储。

# 🔍 output 最终在栈还是堆？

这完全取决于外部如何“安置”这个 Future：

1. 场景一：在父异步函数中直接 .await（通常在栈上）

```rust
async fn parent() -> Result<(), Error> {
    // 直接 .await，Future 作为局部变量内联在 parent 的状态机里
    let future = async {
        let mut reader = Cursor::new([1, 2, 3, 4]);
        let mut output = [0u8; 4];
        reader.read_exact(&mut output).await?;
        Ok::<_, Error>(())
    };
    future.await
}
```

此时 future 这个 Future 对象本身就在 parent() 的状态机里，而 parent() 的状态可能继续在栈上（当它被手动轮询时），所以output 实际上位于栈上（更准确地说，位于某个大结构体内部的栈内存中）。

2. 场景二：通过 tokio::spawn 等执行器任务（在堆上）

```rust
tokio::spawn(async {
    let mut reader = Cursor::new([1, 2, 3, 4]);
    let mut output = [0u8; 4];
    reader.read_exact(&mut output).await?;
    Ok::<_, Error>(())
});
```

tokio::spawn 要求 Future 是 'static 的，并且会在运行时将 Future 放置在堆上（通过 Box::new 类似的方式）。因此，output 现在存在于堆分配的内存中。

3. 场景三：手动 Box::pin（明确在堆上）

```rust
let future = Box::pin(async {
    let mut reader = Cursor::new([1, 2, 3, 4]);
    let mut output = [0u8; 4];
    reader.read_exact(&mut output).await?;
    Ok::<_, Error>(())
});
```

这也会把整个 Future（包括 output）移到堆上。

# 💎 核心结论

- 语义上：output 是 async 块生成的 Future 状态机的一部分，不再是独立的栈变量。
- 物理上：它位于这个 Future 对象内部。而 Future 对象本身：

  + 若被直接 .await 而未装箱，它通常在栈上（由父 Future 的存储决定）。

  + 若通过执行器派发（spawn）或手动装箱，则在堆上。

所以，简单的回答是：不一定在栈上，具体取决于这个异步块是如何被执行的。在常见的 spawn 场景里，它其实在堆上；如果只是在一个局部异步函数里内联使用，它就在栈上。
