`container[index]` 展开后需要加 `*` 的根本原因在于：`Index::index` 返回的是引用，而索引表达式的位置语义需要直接操作值本身。 下面从 trait 设计意图、语法解糖机制、所有权与复制、以及可变上下文四个层面拆解。

---

# 1. 为什么 `Index::index` 返回引用，而不是值？

设想如果 `Index` 被设计成直接返回所有权：

```rust
trait Index<Idx> {
    type Output;
    fn index(&self, idx: Idx) -> Self::Output;
}
```

那么对于 `Vec<String>`，执行 `v[0]` 会将字符串的所有权移出容器。这会导致：

- `v[0]` 之后容器内部该位置变成无效状态，除非用某种标记处理，否则违反内存安全。

- 容器必须拥有元素所有权才能移出，但大部分集合并不希望破坏自身完整性。

因此，标准的 `Index` 被设计为返回不可变引用 `&Self::Output`，`IndexMut` 返回可变引用 `&mut Self::Output`。这样既允许只读访问，又不会夺走所有权，容器始终保持合法状态。

---

# 2. 解糖时为什么要加 `*` —— 将引用“落地”为值

索引表达式被用在值表达式的上下文中，用户期望的是直接拿到元素，而不是一个引用。编译器按照如下规则展开：

```rust
let x = container[index];
// 实际展开为：
let x = *container.index(index);
// 或完全限定语法：
let x = *std::ops::Index::index(&container, index);
```

这里的 `*` 就是对 `&Self::Output` 解引用，得到 `Self::Output` 类型的占位值。

随后，Rust 会按照正常的值使用规则处理这个占位值：

- 如果 `Self::Output` 是 `Copy` 类型（如 `i32`），则 `x` 会得到一个位复制，原容器内的值依旧有效。

- 如果 `Self::Output` 是 非 `Copy` 类型（如 `String`），则尝试移动所有权。但此时会发生编译错误，因为不能通过引用把所有权移出容器。这正是为何 `v[0]` 对于 `Vec<String>` 会报错：`cannot move out of index`。你必须使用 `&v[0]` 或 `v[0].clone()`。

如果去掉这个 `*`，让解糖变成 `let x = container.index(index);`，那么 `x` 就会是一个引用，用户需要手动解引用才能操作值，违背了方括号索引的直觉。

---

# 3. 自动借用与解引用的协调

Rust 的索引表达式还有一个精细之处：它会根据上下文自动进行引用或解引用，这一切都建立在 `*` 已经将引用降级为值占位符的基础上。

例如：

```rust
let v = vec![1, 2, 3];
let a = v[0];          // i32 (Copy) → 复制
let b: &i32 = &v[0];   // 显式借用，等价于 &(*v.index(0))
```

对于 `b`，步骤是：

1. `v.index(0)` 返回 `&i32`。

2. `*` 解引用得到一个 `i32` 占位符（不真正产生移动，因为将立即被借用）。

3. 外层的 `&` 重新取引用，最终 `b` 是 `&i32`。

这个机制允许开发者用极其自然的方式编写代码，而无需关心背后是引用还是值。

---

# 4. 可变上下文：`*` 同样不可或缺

对于写入操作：

```rust
container[index] = value;
// 展开为：
*container.index_mut(index) = value;
```

`index_mut` 返回 `&mut Self::Output`，必须通过 `*` 解引用得到一个位置表达式（place expression），才能将右侧的值赋给这个位置。
如果去掉 `*`，写成 `container.index_mut(index) = value;`，这是不合语法的，因为不能把值赋给一个引用变量本身。

---

# 5. 总结设计逻辑链

1. 安全 → `index()` 必须返回引用，防止所有权被无意移出。

2. 易用 → 方括号语法应该是 `container[idx]` 直接代表元素值，而非引用。

3. 协调 → 编译器在引用返回值和用户期望的值语法之间，自动插入 `*` 进行解引用，并借助 Rust 的值语义（Copy/移动/借用）进一步适配上下文。

因此，`*` 是连接“安全引用返回”与“直觉值操作”的桥梁。它由编译器隐式添加，让程序员在享受所有权安全的同时，保留了容器索引的简洁写法。
