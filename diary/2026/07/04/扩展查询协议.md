# 扩展查询协议

扩展协议允许预编译语句、参数化执行和二进制数据。核心消息如下：

## Parse ('P') —— 解析

```text
'P'
Int32   长度
String  语句名 (null 结尾，空串表示无名语句)
String  SQL 查询 (null 结尾)
Int16   参数类型 OID 数量
Int32[N] 参数类型 OID (0 表示未指定)
```

服务器回复：**ParseComplete ('1')** 或错误。

## Bind ('B') —— 绑定

将已解析的语句与具体参数及结果格式绑定，形成**门户 (Portal)**：

```text
'B'
Int32   长度
String  目标门户名 (null 结尾)
String  源预编译语句名 (null 结尾)
Int16   参数格式码数量（可少于实际参数个数）
Int16[N] 参数格式码 (0 文本 / 1 二进制)
Int16   参数值数量
对每个参数：
  Int32   值长度 (-1 代表 NULL)
  ByteN   值数据 (文本无需 null)
Int16   结果列格式码数量
Int16[N] 结果列格式码
```

回复：**BindComplete ('2')**。

## Describe ('D') —— 描述

获取语句或门户的结果元信息：

```text
'D'
Int32   长度
Byte1   'S' 表示预编译语句，'P' 表示门户
String  名称 (null 结尾)
```

回复：**RowDescription ('T')**（列信息）或 **NoData ('n')**。

## Execute ('E') —— 执行

```text
'E'
Int32   长度
String  门户名 (null 结尾)
Int32   最大返回行数 (0 无限制)
```

服务器可能返回：**DataRow('D')**、**CommandComplete('C')**、**PortalSuspended('s')**。

## Sync ('S')

```text
'S'
Int32   长度 (4)     // 仅长度字段，无消息体
```

要求服务器完成所有待处理命令，回复 **ReadyForQuery('Z')**。

## 完整流程示例

```text
客户端: Parse (P)       → 服务器: ParseComplete (1)
客户端: Bind (B)        → 服务器: BindComplete (2)
客户端: Describe (D)    → 服务器: RowDescription (T)
客户端: Execute (E)     → 服务器: DataRow (D) ... CommandComplete (C)
客户端: Sync (S)        → 服务器: ReadyForQuery (Z)
```

## Flush ('H')

```text
'H'
Int32   长度 (4)
```

强制服务器立即刷新输出缓冲区，常与管道模式配合。
