# 连接建立与启动阶段

客户端首先建立 TCP 连接（默认端口 5432）。随后进入启动协商。

## SSL 协商 (可选)

客户端发送一个 8 字节的特殊消息：

```text
Int32 (8)            - 长度
Int32 (80877103)     - SSL 请求代码
```

服务器收到后回复单字节：

- `'S'`：同意，双方立即开始 SSL/TLS 握手，完成后重发 `StartupMessage`。

- `'N'`：拒绝，客户端继续明文发送 `StartupMessage`。

## GSSAPI 加密协商 (可选)

类似 SSL，代码为 `80877104`。

## StartupMessage

无消息类型码，直接发送：

```text
Int32   总长度 (包含自身)
Int32   协议版本 (3.0 为 196608，即 major=3, minor=0)
[ Key-value 对, 均以 '\0' 结尾 ]
最后以一个 '\0' 结束键值对列表
```

常见键值对：

"user\0用户名\0", "database\0数据库名\0", "application_name\0myapp\0", "options\0...\0"

示例伪数据包：

```text
00 00 00 2E   // 长度 46
00 03 00 00   // 协议 3.0
75 73 65 72 00 70 6F 73 74 67 72 65 73 00   // user\0postgres\0
64 61 74 61 62 61 73 65 00 74 65 73 74 00   // database\0test\0
00          // 结束空字节
```
