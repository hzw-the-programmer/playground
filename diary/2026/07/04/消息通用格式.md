# 消息通用格式

除了一些特殊消息（启动、取消、SSL 请求等），所有消息都遵循：

```text
Byte1   - 消息类型标识 (如 'Q', 'T', 'D', 'C', 'Z')
Int32   - 消息总长度 (包括长度字段本身的 4 字节)
[Payload] - 消息具体内容
```

## 前端 → 后端 (部分消息类型码)

类型码	含义
'p'	PasswordMessage (密码)
'Q'	Query (简单查询)
'P'	Parse (解析预编译语句)
'B'	Bind (绑定参数)
'D'	Describe (描述语句/门户)
'E'	Execute (执行门户)
'S'	Sync (同步点)
'H'	Flush (刷新输出)
'C'	Close (关闭语句/门户)
'd'	CopyData (COPY 数据)
'c'	CopyDone (COPY 完成)
'f'	CopyFail (COPY 失败)
'X'	Terminate (断开连接)

## 后端 → 前端 (部分消息类型码)

类型码	含义
'R'	Authentication (认证请求)
'K'	BackendKeyData (取消密钥)
'S'	ParameterStatus (参数状态)
'Z'	ReadyForQuery (就绪)
'T'	RowDescription (行描述)
'D'	DataRow (数据行)
'C'	CommandComplete (命令完成)
'E'	ErrorResponse (错误)
'N'	NoticeResponse (通知)
'A'	NotificationResponse (异步通知)
'1'	ParseComplete
'2'	BindComplete
'3'	CloseComplete
'n'	NoData
's'	PortalSuspended
'G'	CopyInResponse
'H'	CopyOutResponse
'I'	EmptyQueryResponse
