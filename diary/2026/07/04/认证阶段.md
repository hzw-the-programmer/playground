# 认证阶段

服务器处理 StartupMessage 后，会发送一系列认证和参数消息，最后以 **ReadyForQuery ('Z')** 结束。

## 认证请求 Authentication ('R')

格式：

```text
'R'
Int32   长度
Int32   认证类型码
[根据类型的数据]
```

常见认证类型码：

值	说明
0	AuthenticationOk
2	KerberosV5
3	CleartextPassword
5	MD5Password
6	SCMCredential
7	GSS
8	GSSContinue
9	SSPI
10	SASL
11	SASLContinue
12	SASLFinal

### MD5 认证过程

1. 服务器发送 `'R'` 消息，类型码 5，后跟 4 字节随机盐。

2. 客户端计算：`md5(md5(password + user) + salt)` ，并加上 `"md5"` 前缀。

3. 客户端发送 `PasswordMessage ('p')`：

   ```text
   'p'
   Int32   长度
   String  密码字符串（如 "md5xxxxxxxx"）
   ```

### SCRAM-SHA-256 认证过程 (SASL)

1. 服务器发送 `AuthenticationSASL (10)`，内含多个机制名（`SCRAM-SHA-256\0SCRAM-SHA-256-PLUS\0\0`）。

2. 客户端选择一个机制，发送 **PasswordMessage ('p')**，内容为 SASL 初始响应：`"SCRAM-SHA-256\0<客户端第一消息>"`

3. 服务器回复 `AuthenticationSASLContinue (11)`，携带服务器第一消息（salt, iteration 等）。

4. 客户端计算后发送另一个 **PasswordMessage**，携带客户端最终证明。

5. 服务器回复 `AuthenticationSASLFinal (12)` 包含服务器签名，随后 `AuthenticationOk`。

## ParameterStatus ('S')

服务器推送运行参数，格式：

```text
'S'
Int32   长度
String  参数名 (null 结尾)
String  参数值 (null 结尾)
```

例：`server_version\015.4\0`

## BackendKeyData ('K')

```text
'K'
Int32   长度 (12)
Int32   后端进程 PID
Int32   取消密钥
```

用于后续发送 **取消请求**。

## ReadyForQuery ('Z')

```text
'Z'
Int32   长度 (5)
Byte1   事务状态 ('I' 空闲, 'T' 在事务块中, 'E' 在失败事务块中)
```

表示服务器已准备好接受新的查询。
