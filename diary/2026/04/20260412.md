  FIPS (Federal Information Processing Standards)
  制定方：NIST（美国国家标准与技术研究院）
  适用对象：
  强制：美国联邦机构、政府承包商、处理联邦数据的企业
  自愿：全球金融、医疗、政企、云服务商（AWS/Azure/Google Cloud 均支持 FIPS 模式）
  核心领域：密码算法、密码模块、哈希函数、数据安全、身份认证
  
  NIST (National Institute of Standards and Technology, 美国国家标准与技术研究院) 
  隶属：美国商务部（非军方、非 NSA）
  前身：1901 年成立的 NBS (国家标准局)，1988 年改为 NIST
  总部：马里兰州盖瑟斯堡
  核心使命：制定计量、标准、技术规范，提升美国产业竞争力与国家安全
  非监管机构：不执法、不认证，但FIPS 标准由 NIST 制定，美国政府强制使用

  FIPS 140-3 / 140-2：密码模块安全标准（最核心）
  全称：Security Requirements for Cryptographic Modules
  定义：所有硬件 / 软件 / 固件密码模块（加密芯片、加密库、VPN、HSM、TLS 栈）必须满足的安全要求
  版本：
  FIPS 140-2（2001）：旧版，仍大量在用
  FIPS 140-3（2019）：现行最新标准，替代 140-2，对齐 ISO/IEC 19790
  4 个安全等级（Level 1 ~ Level 4）：
  Level 1：基础软件加密（如 OpenSSL FIPS 模块）
  Level 2：防篡改、角色认证（如加密硬盘）
  Level 3：防物理攻击、密钥零化（如 HSM、加密机）
  Level 4：最高等级，防环境攻击（极端温度 / 电压）

  FIPS 197：高级加密标准（AES）
  对应算法：AES (Rijndael)
  规格：128/192/256 位密钥，128 位分组
  地位：全球最主流对称加密标准，替代旧 DES (FIPS 46)
  合规要求：FIPS 场景下只能用 AES，禁止 DES/3DES（已废弃）

  FIPS 180-4：安全哈希标准（SHS）
  对应算法：SHA-1、SHA-224、SHA-256、SHA-384、SHA-512
  关键：
  SHA-1：已不安全，FIPS 场景禁止用于签名
  SHA-256 / SHA-512：当前合规主力
  
  FIPS 202：SHA-3 标准
  对应算法：SHA-3 (Keccak)
  定位：新一代抗碰撞哈希，作为 SHA-2 备用

  FIPS 203/204/205：抗量子密码标准（最新）
  2024 年 8 月发布
  FIPS 203 (ML-KEM)：密钥封装（替代 ECDH/RSA）
  FIPS 204 (ML-DSA)：数字签名（替代 ECDSA/RSA）
  FIPS 205 (SLH-DSA)： stateless 哈希签名
  意义：后量子时代 FIPS 合规必备

  Diffie–Hellman key exchange（DH 密钥交换）
  一句话：让双方在明文信道上，安全协商出一个只有彼此知道的共享密钥，全程不传递密钥本身。
  1. 核心思想（极简数学）
  公开一个大素数 p 和一个原根 g（所有人都能看）
  A 选私钥：a
  A 发公钥：A = g^a mod p
  B 选私钥：b
  B 发公钥：B = g^b mod p
  双方计算共享密钥
  A 算：B^a mod p = (g^b)^a mod p = g^(ab) mod p
  B 算：A^b mod p = (g^a)^b mod p = g^(ab) mod p
  结果一样 → 这就是共享密钥
  攻击者只能看到：p、g、A、B
  但没有 a /b，算不出 g^(ab) mod p

  2. 为什么安全？
  基于离散对数难题：
  已知 g、p、g^x mod p，很难反推出 x

  3. 现实中不用普通 DH，用 ECDH
  Elliptic Curve Diffie–Hellman
  椭圆曲线版 DH
  密钥更短（256 bit ≈ RSA 3072）
  更快、流量更小
  TLS 1.3 里全部用 ECDHE

  ECDHE 流程（和 DH 完全一样，只是数学换成椭圆曲线）
  客户端生成：私钥 x / 公钥 X
  服务端生成：私钥 y / 公钥 Y
  交换公钥
  双方算：
  client：x * Y
  server：y * X
  → 得到相同共享密钥（shared secret）

  DHE = Diffie-Hellman Ephemeral
      = 临时 Diffie-Hellman 密钥交换算法
  和你刚才看的 DH 区别就一个词：Ephemeral（临时、一次性）

  1. DHE 是什么意思？
  Diffie-Hellman
  Ephemeral（临时的）
  每次 TLS 握手，都重新生成一对全新的 DH 公私钥，用完即弃。

  2. 为什么要 Ephemeral？
  为了 PFS（Perfect Forward Secrecy，完美前向保密）
  如果服务器长期私钥泄露
  但 DHE 每次密钥都是临时生成、不留存
  攻击者即使截获历史密文，也无法解密过去的会话

  这就是现代 TLS 1.3 强制使用 ECDHE/DHE 的原因。

  4. DHE 算法流程（极简）
  双方约定 DH 参数：(p, g)
  客户端生成临时私钥 a → 计算公钥 A = g^a mod p
  服务端生成临时私钥 b → 计算公钥 B = g^b mod p
  交换 A、B
  各自计算：
  客户端：B^a mod p
  服务端：A^b mod p
  得到相同 共享密钥 Z（就是你之前的 IKM）

  5. 为什么现在很少直接用 DHE？
  因为模幂运算非常慢，尤其大素数 p 很大时。
  所以现代 TLS 都用：
  ECDHE = Elliptic Curve Diffie-Hellman Ephemeral
  密钥小（256bit 就够）
  计算快 1~2 个数量级
  安全性更高
