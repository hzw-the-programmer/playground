# 原子内存顺序：`Acquire` / `Release` / `AcqRel` 完整拆解

前置基础：CPU 会乱序执行读写，内存屏障用来限制重排、保证跨线程数据可见性。

原子操作分三类：load（读）、store（写）、compare_exchange（读 + 写复合 CAS）。

# 一、`Release`（只用于 store / CAS 成功分支）

核心规则

1. 本线程所有发生在 Release store 之前的读写，绝对不能重排到 store 之后；
2. 其他线程用 Acquire load 读取这个原子变量时，能完整看到本次 store 之前全部写入的数据。

通俗比喻

Release = 发布公告

你先把货物（业务数据）全部装好，再贴公告（更新原子索引）；别人看到公告，就一定能拿到完整货物，不会拿到半成品。

对应 Tokio 队列例子

```rust
self.inner.tail.store(new_tail, Release);
```

流程：

1. 先把任务写入 buffer（业务数据）；
2. 再执行 Release store 更新 tail；
3. 窃取线程执行 tail.load(Acquire)，只要读到新 tail，必然能读到完整初始化的任务，不会读到 MaybeUninit 垃圾内存。

限制

Release 只能阻止写往后飘，无法阻止读往前飘；不能单独用来读。

# 二、`Acquire`（只用于 load / CAS 失败分支）

核心规则

1. 本线程所有发生在 Acquire load 之后的读写，绝对不能重排到 load 之前；
2. 如果读到了别的线程通过 Release store 写入的值，就能同步看到对方 Release 之前所有写入。

通俗比喻

Acquire = 查看公告

看到公告（原子变量更新），就可以放心去取公告对应的货物，保证货物已经全部存放完毕。

对应 Tokio 队列例子

窃取线程读取队尾：

```rust
let src_tail = self.0.tail.load(Acquire);
```

1. 读取 `tail` 之后，访问 `buffer` 读取任务，不会被 CPU 重排到读取 `tail` 之前；
2. 只要读到更新后的 `tail`，生产者之前写入缓冲区的任务全部对当前线程可见。

限制

Acquire 只能阻止读往前飘，无法阻止写往后飘；不能单独用来写。

# 三、AcqRel（仅用于 CAS 成功分支，读改写复合序）

CAS 是「先 load 读、再 store 写」的复合操作，`AcqRel` = Acquire + Release 叠加

规则拆分

1. 读侧（Acquire 能力）：执行 CAS 读取旧值时，拥有 Acquire 语义，同步其他线程所有 Release 写入；
2. 写侧（Release 能力）：CAS 修改成功、写入新值时，拥有 Release 语义，把本次修改同步给其他线程。

通俗比喻

AcqRel = 一边看公告、一边贴新公告

先看旧公告（Acquire，同步别人的数据）；

修改完状态，再贴新公告（Release，同步自己的修改给所有人）。

Tokio 队列场景

`pop()` / `steal_into2` 的核心 CAS：

```rust
head.compare_exchange_weak(old, new, AcqRel, Acquire)
```

1. Acquire：读取旧的复合表头 `(steal, real)`，同步生产者、其他窃取者对队列的所有修改；
2. Release：更新新表头后，其他线程执行 `load(Acquire)` 能立刻感知队列状态变化（比如正在窃取标记、队头偏移）。

使用场景限定

只有同时需要读、又需要对外发布修改的共享原子变量才用 AcqRel；单纯 store 不用，单纯 load 不用。

# 四、三者配对关系（必须成对才生效）

1. Release store ↔ Acquire load 是唯一能跨线程同步数据的组合
   - 单方面 Release / 单方面 Acquire 没有任何同步效果；

2. AcqRel 内部自带 Release+Acquire，天然可以和任意 Acquire load 配对。

# 五、极简区分速记

1. Release：写专用，写完数据再更新原子，对外发布；
2. Acquire：读专用，读到原子再读取业务数据，同步别人的修改；
3. AcqRel：CAS 成功专用，先读同步别人、后写发布自己，双向同步。

# 六、补充：和代码里 Relaxed 对比

Relaxed 无任何重排限制，仅保证原子变量自身读写原子，不做任何跨线程数据同步。

- Release/Acquire/AcqRel 是带同步屏障的强内存序；
- Relaxed 是最弱内存序，仅用于无需同步业务数据的场景。
