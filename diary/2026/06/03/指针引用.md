在 Rust 中，指针通常特指裸指针（`*const T` 和 `*mut T`），而引用（`&T` 和 `&mut T`）是更安全、更常用的间接访问方式。它们的核心区别围绕安全性、编译期保证和表达能力展开。

下面从多个维度详细拆解。

---

# 1. 安全性与借用检查

引用：100% 安全（Safe Rust）。编译器通过借用检查器强制执行严格的别名规则：

- `&T`：可以有多个共享引用，它们都只读，不修改原数据。

- `&mut T`：同一时刻只能有一个可变引用，在此期间不能有任何共享引用。

这些规则在编译期杜绝了数据竞争和悬垂引用。

指针：本身就是 `unsafe` 的。解引用（读取或修改）裸指针必须在 `unsafe` 块内进行。编译器不检查别名关系，一个 `*mut T` 和另一个 `*const T`（甚至另一个 `*mut T`）可以同时指向同一块内存。正确性完全由程序员保证。

# 2. 生命周期与悬垂

- 引用：总是携带隐式的生命周期参数（可被省略），编译器会检查它是否比被引用的数据活得长。无法创建悬垂引用（除非你用 `unsafe` 自己骗编译器）。

  ```rust
  let r;
  {
      let x = 5;
      r = &x; // 编译错误：x 的生命周期不够长
  }
  ```

- 指针：没有生命周期信息。它可以轻松变成悬垂指针（dangling pointer），编译器对此毫无意见。

  ```rust
  let p: *const i32;
  {
      let x = 5;
      p = &x as *const i32; // 可行，x 被销毁后 p 悬垂
  }
  // p 现在悬垂，解引用是未定义行为
  ```

# 3. 空值与有效性保证

- 引用：不允许为空。Rust 在语言层面保证 `&T` 和 `&mut T` 总是非空的。这一保证使编译器能进行布局优化（如 `Option<&T>` 大小与指针一致），并省去大量空值检查。

- 指针：可以为空（`std::ptr::null()` 或 `std::ptr::null_mut()`）。解引用空指针导致未定义行为。

# 4. 所有权与 Copy

- 引用：

  + `&T`：实现 `Copy`，因为同时存在多个只读引用是安全的。它不涉及所有权转移。

  + `&mut T`：不实现 `Copy`（也不能 `Clone`），因为独占性是它的核心。复制可变引用会违反“唯一可变引用”规则。移动 `&mut T` 时会转移这个独占借用。

- 指针：无论是 `*const T` 还是 `*mut T`，都实现 `Copy`。指针只是一个地址值，按位复制是平凡的，所有权和借用规则都不适用。

# 5. 解引用与自动行为

- 引用：在许多上下文中可以自动解引用。例如通过 `.` 操作符可以直接调用 `T` 的方法，赋值时会自动配合解引用强制多态（deref coercion）。

  ```rust
  let x: i32 = 5;
  let r: &i32 = &x;
  println!("{}", r.abs()); // 自动解引用，等价于 (*r).abs()
  ```

- 指针：必须显式、非安全地解引用，且必须在 `unsafe` 块内。

  ```rust
  let p: *const i32 = &x;
  unsafe {
      println!("{}", (*p).abs()); // 必须显式 *p
  }
  ```

  解引用时没有任何边界检查或有效性检查。

# 6. 类型上的 trait 实现差异

- 引用：根据 `T` 是否满足某些条件，`&T`（以及 `&mut T`）会自动实现 `Send`、`Sync`、`UnwindSafe` 等标记 trait，非常符合直觉。例如，如果 `T: Sync`，则 `&T: Send`，因为共享引用可以被安全地发送到其他线程。

- 指针：不实现任何自动 trait。`*const T` 和 `*mut T` 既不 `Send` 也不 `Sync`。跨线程传输裸指针是安全的（只是复制地址），但将裸指针包装成可安全跨线程的抽象需要手动用 `unsafe impl Send for ...` 保证，完全把责任交给程序员。

# 7. 运算与偏移

- 引用：你无法对引用做算术运算（如 `r + 1`）。要访问内存块中偏移位置的数据，通常通过切片（`&[T]`）和迭代器，或者先转换为指针。

- 指针：带有 `add()`、`sub()`、`offset()`、`wrapping_add()` 等方法，可以做任意指针算术。这是实现数据结构（如自定义 `Vec`）或与 C 交互时所必需的。

# 8. 创建与获取方式

- 引用：

  + 使用 `&` 或 `&mut` 操作符从已有变量借用。

  + 如果实现了 `Deref`，也可以从智能指针（如 `Box<T>`）获取引用。

  + 永远不能从凭空捏造的数字创建引用（那属于裸指针的范畴）。

- 指针：

  + 从引用强制转换：`let p = &x as *const i32;`

  + 从智能指针获取原始指针：`Box::into_raw(b)` 或 `&*boxed`。

  + 使用 `std::ptr::addr_of!` 和 `std::ptr::addr_of_mut!` 宏获取某个字段的指针（即使该变量未被正确初始化）。

  + 从具体的地址字面量构建（如 FFI 返回的地址，或内存映射 I/O）：`let p = 0x4000 as *const u32;`。

# 9. 布局与 ABI

在编译后，引用和指针都具有相同的内存表示：就是一个机器字长的内存地址。它们可以安全地互相转换（`as`）。调用 C 函数时，`&T` 会被编译为和 `*const T` 相同的指针传递，所以 FFI 中能直接传入引用（前提是遵守安全性规则）。

# 总结：一张表看懂

特性	引用 (`&T` / `&mut T`)	裸指针 (`*const T` / `*mut T`)
安全性	安全（Safe Rust）	`unsafe`，解引用需 `unsafe` 块
借用检查	严格，保证无数据竞争/悬垂	无，可随意别名、悬垂
生命周期	有，编译器追踪	无，程序员自行管理
空值	不允许	允许
所有权模型	借用（不拥有数据）	不拥有，也不遵守借用规则
`Copy` 实现	`&T` 是 `Copy`；`&mut T` 不是	全是 `Copy`
解引用	自动解引用（`.`操作符）	必须显式 `*ptr`，`unsafe`
指针算术	不支持（切片/索引除外）	支持 `add`/`sub`/`offset`
自动 Trait	继承 `T` 的 `Send`/`Sync` 等	不实现任何自动 trait
创建方式	`&` / `&mut` 借用	转换自引用、`Box::into_raw`、地址直接构造
典型用途	日常安全编程	FFI、底层数据结构、性能优化

一句话概括：引用是对数据的安全、受约束的观察口，由编译器担保正确；裸指针是无保护的原始内存地址，灵活性最大，但安全性完全由程序员承担。
